Create lib/record-context.tsx as a 'use client' context provider.

The context must store ALL state shared across the /record route family:
- serial: string — UUID generated at session start (crypto.randomUUID())
- discipline: 'long-cycle' | 'jerk' | 'snatch' | null
- disciplineLabel: string | null — human-readable ("10 Min. Long Cycle", "10 Min. Jerk", "10 Min. Snatch")
- athleteName: string — fetched from Clerk and passed in
- weightKg: number | null
- countdownSeconds: number — default 10
- beepEveryMinute: boolean — default false
- autoStop: boolean — default false
- selectedDeviceId: string | null
- recordedBlob: Blob | null
- mimeType: string — detected MIME type (e.g. 'video/webm' or 'video/mp4')

Setters for each piece of state.

Export:
- RecordProvider — wraps children, generates serial on mount
- useRecord — hook that throws if used outside provider

TypeScript interface RecordState with all fields + setters.

Do NOT put athlete name in initial state — it is passed as a prop to RecordProvider and stored in context so routes can read it without server calls.

Create app/record/page.tsx. This is a CLIENT component ('use client').

STEP A — iOS/unsupported browser gate:
At the top of the component, detect browser support using useEffect + useState:
```tsx
const [supported, setSupported] = useState<boolean | null>(null) // null = loading

useEffect(() => {
  // canvas.captureStream is the key API needed
  const canvas = document.createElement('canvas')
  setSupported(typeof canvas.captureStream === 'function')
}, [])
```

If supported === null: show nothing (or a minimal loading state)
If supported === false: show full-screen unsupported message:
  - Dark background (bg-zinc-950), centered content
  - Title: "Browser Not Supported"
  - Message: "Kova recorder requires Chrome on Android or a desktop browser. iOS is not supported."
  - A "Back to Dashboard" link → href="/dashboard"
  - Style: clean, no panic, informative

STEP B — Discipline selection (shown only when supported === true):
Three full-width cards in a flex column layout, mobile-first, full screen:
