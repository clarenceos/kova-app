<objective>
Add YouTube auto-upload to the recording flow. After recording, athlete taps "Upload to YouTube" on the playback page. The video uploads directly from the browser to YouTube Data API v3 using a resumable upload with progress tracking. On completion, the entry is saved to DB with youtube_url, youtube_id, and status='pending'.

Purpose: Eliminates manual YouTube upload — the biggest friction point between recording and getting judged.
Output: YouTube upload server action, uploader component, updated playback and instructions pages.
</objective>

1. Create `lib/actions/youtube.ts` with `'use server'` directive:
```typescript
'use server'

import { clerkClient, auth } from '@clerk/nextjs/server'

export async function getYouTubeToken(): Promise<{ token: string } | { error: string }> {
  const { userId } = await auth()
  if (!userId) return { error: 'Unauthorized' }

  try {
    const client = await clerkClient()
    const tokens = await client.users.getUserOauthAccessToken(userId, 'oauth_google')

    if (!tokens.data || tokens.data.length === 0) {
      return { error: 'google_not_connected' }
    }

    return { token: tokens.data[0].token }
  } catch (err) {
    console.error('[getYouTubeToken] Failed:', err)
    return { error: 'Failed to retrieve Google token' }
  }
}
```

2. Create `components/record/YouTubeUploader.tsx` — a `'use client'` component that:

**Props interface:**
```typescript
interface YouTubeUploaderProps {
  blob: Blob
  mimeType: string
  title: string         // e.g. "KOVA | 10 Min. Long Cycle | 24KG | John Smith | ABC-1234"
  description: string   // from buildYouTubeDescription
  serial: string
  athleteName: string
  discipline: string    // underscored format: 'long_cycle'
  weightKg: number
  onUploadComplete: (youtubeUrl: string, youtubeId: string) => void
}
```

**States:**
- `status`: `'idle' | 'authenticating' | 'uploading' | 'saving' | 'complete' | 'error' | 'google_not_connected'`
- `progress`: number (0-100)
- `errorMessage`: string
- `youtubeUrl`: string | null

**Upload flow (triggered by button click):**

Step 1 — Get token:
- Set status to 'authenticating'
- Call `getYouTubeToken()` server action
- If error is 'google_not_connected': set status to 'google_not_connected', show message "Connect your Google account to upload directly to YouTube" with a link/button. The link should point to the Clerk user profile page or use `window.Clerk?.openSignIn()` — but simplest approach: show text instructing user to go to their account settings. Use `<a href="/user-profile">Connect Google Account</a>` (Clerk's default user profile route).
- If other error: set status to 'error', show error message

Step 2 — Initiate resumable upload:
- Set status to 'uploading', progress to 0
- POST to `https://www.googleapis.com/upload/youtube/v3/videos?uploadType=resumable&part=snippet,status`
- Headers: `Authorization: Bearer ${token}`, `Content-Type: application/json`, `X-Upload-Content-Length: ${blob.size}`, `X-Upload-Content-Type: ${mimeType}`
- Body JSON:
```json
{
  "snippet": {
    "title": title,
    "description": description,
    "tags": ["kova", "kettlebell", discipline.replace('_', ' '), "competition"],
    "categoryId": "17"
  },
  "status": {
    "privacyStatus": "unlisted"
  }
}
```
- Extract `Location` header from response — this is the upload URI
- If response not 200: set error with response status text

Step 3 — Upload blob with progress:
- Use XMLHttpRequest (not fetch) for progress tracking via `xhr.upload.onprogress`
- PUT to the upload URI from step 2
- Headers: `Content-Type: ${mimeType}`
- Body: the blob directly
- On progress: update `progress` state as `Math.round((event.loaded / event.total) * 100)`
- On complete (status 200): parse response JSON, extract `id` field as videoId
- Construct `youtubeUrl = https://youtube.com/watch?v=${videoId}`

Step 4 — Save entry to DB:
- Set status to 'saving'
- Call `createEntry({ athleteName, discipline, weightKg, serial, youtubeUrl, youtubeId: videoId })` server action
- If error: set status to 'error' with message
- On success: set status to 'complete', set youtubeUrl state, call `onUploadComplete(youtubeUrl, videoId)`

**Retry:** On error state, show "Retry" button that restarts from step 1.

**UI layout:**
- When idle: large "Upload to YouTube" button (patina-bronze bg, same style as existing export button)
- When authenticating: button disabled, text "Connecting to YouTube..."
- When uploading: progress bar (div with width percentage, patina-bronze bg) + percentage text + "Uploading to YouTube..." text. Estimated time display not required (nice-to-have, skip if complex).
- When saving: "Saving entry..." spinner
- When complete: success card showing serial prominently, "Upload Complete" with green check, youtubeUrl as a link, "View on YouTube" link opening in new tab
- When google_not_connected: message card with "Connect your Google account to upload" + link to connect + "Skip — upload manually" button that navigates to /record/instructions
- When error: error message in raw-steel text + "Retry" button

Use Tailwind classes consistent with existing app styling: `bg-forge-black`, `bg-charcoal`, `text-parchment`, `text-raw-steel`, `bg-patina-bronze`, `rounded-xl`, `rounded-2xl`.
