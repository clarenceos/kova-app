1. Create `components/profile/GhostReplay.tsx` — a `'use client'` component:

**Purpose:** Wraps YouTubeEmbed with a ghost replay overlay that shows Check/X icons at the timestamps where the judge tapped rep/no-rep.

**Props:**
```typescript
interface GhostReplayProps {
  videoId: string
  repTaps: Array<{ time: number | null; type: 'rep' | 'no-rep' }>
}
```

**Implementation:**
- Import `YouTubeEmbed` from `@/components/judge/YouTubeEmbed`
- Import `Check`, `X` from `lucide-react`
- State: `playerRef` (useRef for YT player instance), `activeIndicator` (useState: `{ type: 'rep' | 'no-rep'; key: number } | null`)
- Ref: `lastProcessedIndexRef` (useRef<number>(0)) — tracks which taps have been triggered
- Ref: `animFrameRef` (useRef<number>) — for rAF cleanup

**Replay polling loop (started when player is ready):**
```typescript
function pollReplay() {
  if (!playerRef.current) return
  const currentTime = playerRef.current.getCurrentTime() as number

  // Handle seek: if currentTime is before last processed tap, reset index
  const lastIdx = lastProcessedIndexRef.current
  if (lastIdx > 0 && repTaps[lastIdx - 1]?.time !== null && currentTime < (repTaps[lastIdx - 1].time! - 0.5)) {
    // Seek backward detected — find correct index
    let newIdx = 0
    for (let i = 0; i < repTaps.length; i++) {
      if (repTaps[i].time !== null && repTaps[i].time! <= currentTime) {
        newIdx = i + 1
      } else {
        break
      }
    }
    lastProcessedIndexRef.current = newIdx
  }

  // Process taps
  while (lastProcessedIndexRef.current < repTaps.length) {
    const tap = repTaps[lastProcessedIndexRef.current]
    if (tap.time !== null && tap.time <= currentTime) {
      triggerIndicator(tap.type)
      lastProcessedIndexRef.current++
    } else {
      break
    }
  }

  animFrameRef.current = requestAnimationFrame(pollReplay)
}
```

**triggerIndicator(type):**
- Set `activeIndicator` to `{ type, key: Date.now() }` (key forces re-render for animation)
- After 600ms timeout, set `activeIndicator` to null

**onPlayerReady handler:**
- Store player in `playerRef`
- Start polling: `animFrameRef.current = requestAnimationFrame(pollReplay)`

**Cleanup in useEffect return:**
- `cancelAnimationFrame(animFrameRef.current)`

**Render:**
```tsx
<div className="relative">
  <YouTubeEmbed videoId={videoId} onPlayerReady={handlePlayerReady} />

  {/* Ghost replay overlay */}
  {activeIndicator && (
    <div
      key={activeIndicator.key}
      className="pointer-events-none absolute inset-0 flex items-center justify-center animate-ghost-fade"
    >
      {activeIndicator.type === 'rep' ? (
        <Check className="h-20 w-20 text-patina-bronze drop-shadow-lg" strokeWidth={3} />
      ) : (
        <X className="h-20 w-20 text-raw-steel drop-shadow-lg" strokeWidth={3} />
      )}
    </div>
  )}
</div>
```

**CSS animation:** Add a `@keyframes ghost-fade` animation via Tailwind's `animate-` utility. Since Tailwind v4, custom animations can be defined in `app/globals.css` or inline via `style` prop. Simplest approach — use inline styles on the indicator div:
```tsx
style={{ animation: 'ghost-fade 600ms ease-out forwards' }}
```
And add to `app/globals.css` (if not already customizable — check first):
```css
@keyframes ghost-fade {
  0% { opacity: 1; transform: scale(1); }
  100% { opacity: 0; transform: scale(1.3); }
}
```
If Tailwind v4 supports arbitrary animation in the config, use that instead. Fallback: just use the inline style approach, which works regardless of config.
