  <action>
    Create `lib/actions/competitions.ts` with a single exported async function `createCompetition`.

    The function signature:
    ```typescript
    export async function createCompetition(input: {
      name: string
      date: string
      numPlatforms: number
      status: 'draft' | 'open'
      maxRegistrants: number | null
      registrationDeadline: string | null
      allowedDurations: 'both' | '10' | '5'
      allowedBellWeights: string[]
    }): Promise<{ id: string } | { error: string }>
    ```

    Implementation:
    1. File starts with `'use server'`
    2. Import `db` from `@/lib/db`, `competitions` from `@/lib/schema`, `deriveSerialPrefix` from `@/lib/queue/serial`, `revalidatePath` from `next/cache`
    3. Validate required fields:
       - `name` must be non-empty trimmed string -> error: "Competition name is required"
       - `date` must be non-empty string -> error: "Date is required"
       - `numPlatforms` must be integer between 1 and 10 -> error: "Enter a value between 1 and 10"
       - `allowedBellWeights` must be array with length >= 1 -> error: "Select at least one bell weight"
       - `allowedDurations` must be one of 'both', '10', '5' -> error: "Invalid duration rule"
    4. Derive serial prefix: `const serialPrefix = deriveSerialPrefix(input.name.trim())`
    5. Insert into DB:
       ```typescript
       const [row] = await db.insert(competitions).values({
         name: input.name.trim(),
         date: input.date,
         numPlatforms: input.numPlatforms,
         status: input.status,
         maxRegistrants: input.maxRegistrants || null,
         registrationDeadline: input.registrationDeadline || null,
         allowedDurations: input.allowedDurations,
         allowedBellWeights: JSON.stringify(input.allowedBellWeights),
         serialPrefix,
         createdAt: new Date(),
       }).returning({ id: competitions.id })
       ```
    6. Call `revalidatePath('/organizerdb')` after successful insert
    7. Return `{ id: row.id }`
    8. Wrap everything in try/catch, catch returns `{ error: 'Failed to create competition. Please try again.' }`

    Per COMP-02 (D-07): serial prefix is derived server-side and validated against what the client preview showed. The server is the source of truth for prefix derivation — if they diverge, the server value wins.
