## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 400 | 1.4 |
| Heading | 24px | 700 | 1.2 |
| Caption / helper | 12px | 400 | 1.4 |

Notes:
- Body (16px / 400) — form input values, card descriptions, list copy
- Label (14px / 400) — form field labels, section subheadings within the form, badge text; size distinction from Body (16px) provides hierarchy without a separate weight
- Heading (24px / 700) — page title ("Create Competition", "Your Competitions")
- Caption (12px / 400) — serial prefix preview ("Serial prefix: GPC"), helper text under optional fields, inline validation errors

Source: Matches `JudgeSetupForm.tsx` (`text-sm` labels, `text-2xl font-bold` heading). Weight collapsed from 3 to 2 (400, 700) — Label uses size (14px vs 16px) for hierarchy instead of a separate medium weight.

## Layout Contract

### `/organizerdb/create` — Competition Creation Form

- Desktop-first, single-column, centered (source: QUEUE_SPEC R2, CONTEXT.md D-06)
- Max width: 640px, centered with `mx-auto`
- Outer padding: `px-8 py-12` on desktop
- Form container: `bg-card rounded-xl ring-1 ring-foreground/10 p-8` (matches shadcn Card)
- Sections within the form separated by `mb-6` (24px)

Field order (locked — source: CONTEXT.md D-06, QUEUE_SPEC):
1. Competition Name (text input, required) + serial prefix preview beneath
2. Date (date input, required)
3. Number of Platforms (number input, min 1 max 10, default 3)
4. Status (radio group: Draft / Open, default Draft)
5. Max Registrants (number input, optional) — helper: "Leave blank for unlimited"
6. Registration Deadline (datetime-local input, optional) — helper: "Leave blank for no deadline"
7. Duration Rule (radio group: "Both 5 and 10 min" / "10 min only" / "5 min only")
8. Bell Weights — two checkbox sections (see below)
9. Submit button

Bell Weights section layout:
- Section heading: "Allowed Bell Weights" at label size (14px / 400)
- Two subsections: "Double Bell — LC & Jerk" and "Single Bell — Snatch"
- Each subsection: heading + "Select All / Deselect All" link at right + checkbox grid
- Checkbox grid: 3 columns on desktop, weights displayed as `2x8`, `2x12`, etc.
- All checked by default (source: CONTEXT.md D-06)

Serial prefix preview (source: CONTEXT.md D-02):
- Appears directly below the Competition Name input
- Copy: "Serial prefix: GPC" where GPC updates live as user types
- Color: `--color-bright-bronze` (#C49A5A) — accent
- Size: 12px / 400 (caption role)
- Hidden when name field is empty

### `/organizerdb` — Competition List Page

- Desktop-first, full-width with centered content area (max-width: 1024px)
- Page header row: "Your Competitions" heading (left) + "New Competition" button (right)
- Competition cards: `Card` component from shadcn, stacked list (not grid)
- Each card shows: competition name (CardTitle), date + status badge (CardDescription), registration link row (CardContent), serial prefix as small muted caption

Card anatomy:
- Header: name + status badge (Draft/Open)
- Body: date, platform count, registration link with "Copy Link" button
- No footer in this phase (Phase 10 adds analytics bar)

Empty state (no competitions yet): centered in content area, below page header.

## Interaction States

### Input fields
- Default: `border border-raw-steel/30 bg-charcoal` (matches `JudgeSetupForm.tsx`)
- Focus: `focus:border-patina-bronze focus:outline-none transition-colors`
- Error: `border-destructive` (raw-steel, used for inline validation)
- Disabled: `opacity-50 cursor-not-allowed`

### Submit button
- Default: `bg-patina-bronze text-parchment font-bold rounded-2xl py-3 w-full`
- Hover: `hover:bg-bright-bronze`
- Active: `active:opacity-80`
- Loading: `disabled:cursor-not-allowed disabled:opacity-50` + label changes to "Creating..."

### Copy Link button (competition cards)
- Variant: ghost or outline, small size
- After copy: label briefly changes to "Copied!" (1.5s) then reverts to "Copy Link"
- Icon: `Clipboard` from Lucide, 16px, inline left of label

### Checkbox (bell weights)
- Checked: `bg-patina-bronze border-patina-bronze`
- Unchecked: `border-raw-steel/30 bg-charcoal`
- All-checked validation error: displays "Select at least one bell weight" in caption style below the section (source: CONTEXT.md D-05)

### Status badge
- Draft: `bg-raw-steel/20 text-raw-steel` border-less pill
- Open: `bg-bright-bronze/20 text-bright-bronze` border-less pill
