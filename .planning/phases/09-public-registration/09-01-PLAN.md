import { db } from '@/lib/db'
import { competitions, registrants, registrationEntries } from '@/lib/schema'

**Function signature:**
```typescript
export async function registerAthlete(input: {
  competitionId: string
  lastName: string
  firstName: string
  gender: 'Male' | 'Female'
  bodyWeightKg: number
  country: string
  club: string | null
  coach: string | null
  events: Array<{
    event: 'LC' | 'JERK' | 'SNATCH'
    bellWeight: string
    duration: number
  }>
}): Promise<{ registrantId: string } | { error: string }>
```

**Validation (in this order):**
1. `competitionId` must be a non-empty string
2. Fetch competition from DB: `const [comp] = await db.select().from(competitions).where(eq(competitions.id, input.competitionId))`
3. If comp not found: return `{ error: 'Competition not found' }`
4. If `comp.status !== 'open'`: return `{ error: 'Registration is closed' }`
5. If `comp.registrationDeadline` is set and `new Date(comp.registrationDeadline) < new Date()`: return `{ error: 'Registration is closed' }`
6. If `comp.maxRegistrants` is set: count registrants for this competition (`db.select({ count: count() }).from(registrants).where(eq(registrants.competitionId, input.competitionId))`). If count >= maxRegistrants: return `{ error: 'This competition is full' }`
7. `lastName` must be non-empty trimmed string
8. `firstName` must be non-empty trimmed string
9. `gender` must be 'Male' or 'Female'
10. `bodyWeightKg` must be a positive number
11. `country` must be non-empty trimmed string
12. `events` must be a non-empty array
13. Each event must have valid `event` ('LC', 'JERK', 'SNATCH'), non-empty `bellWeight`, and `duration` (5 or 10)

**Atomic insert using db.batch() (per Phase 6 D-08, REG-05):**
1. Generate a cuid2 ID for the registrant: `const registrantId = createId()`
2. Generate serial numbers BEFORE the batch — call `generateCompetitionSerial(input.competitionId, comp.serialPrefix)` once per event entry. Store in an array.
3. Build the batch:
   ```typescript
   const now = new Date()
   await db.batch([
     db.insert(registrants).values({
       id: registrantId,
       competitionId: input.competitionId,
       lastName: input.lastName.trim(),
       firstName: input.firstName.trim(),
       gender: input.gender,
       bodyWeightKg: input.bodyWeightKg,
       country: input.country.trim(),
       club: input.club?.trim() || null,
       coach: input.coach?.trim() || null,
       createdAt: now,
     }),
     ...input.events.map((evt, i) =>
       db.insert(registrationEntries).values({
         registrantId,
         competitionId: input.competitionId,
         event: evt.event,
         bellWeight: evt.bellWeight,
         duration: evt.duration,
         serial: serials[i],
         createdAt: now,
       })
     ),
   ])
   ```
4. Call `revalidatePath('/organizerdb')` after successful insert.
5. Return `{ registrantId }`.

**Error handling:**
Wrap entire function body in try/catch. On catch: `console.error('[registerAthlete] Failed:', msg, error)` and return `{ error: 'Registration failed. Please try again.' }`.

Also export a `getRegistrationData` function for the success page (per D-07):
```typescript
export async function getRegistrationData(registrantId: string): Promise<{
  registrant: Registrant
  entries: RegistrationEntry[]
  competition: Competition
} | null>
```

This function:
1. Fetches registrant by ID from `registrants` table
2. If not found, returns null
3. Fetches all `registrationEntries` where `registrantId` matches
4. Fetches competition by `registrant.competitionId`
5. Returns the assembled object (or null if competition missing)

This is a plain async function (not a server action) — it does NOT need the `'use server'` export since it will be called from a server component directly. But since the file has `'use server'` at the top, all exports become server actions. That is fine — server actions can be called from server components too.
