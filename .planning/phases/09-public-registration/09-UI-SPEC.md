## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline checkbox-label gap (`gap-1`) |
| sm | 8px | Compact element spacing, field helper text margin-top (`mt-2`) |
| md | 16px | Default element spacing, form field gap (`space-y-4`), section padding (`px-4`) |
| lg | 24px | Section padding, card interior padding (`p-6`), header bottom margin (`mb-6`) |
| xl | 32px | Layout gaps, page horizontal padding (`px-8`) |
| 2xl | 48px | Major section breaks, page top padding (`py-12`) |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Form card uses `p-8` (32px) interior padding — matching existing `app/organizerdb/create/page.tsx` card pattern
- Event config expand area uses `pt-4 pb-4 pl-6` (inset under checkbox) to visually nest under the parent event; `pl-6` handles horizontal nesting, `pt-4` (16px) provides standard vertical clearance from the checkbox
- Form `space-y-6` between primary field groups (matching competition creation form)

Accent (`patina-bronze`) reserved for:
- Primary submit button background ("Register" CTA)
- Input focus border (`focus:border-patina-bronze`)
- Focus ring

Bright-bronze reserved for:
- Assigned serial number values on success page
- Helper/preview text (e.g. country flag + name in combobox trigger)
- "Register another athlete" link/button hover

Guard state badge colors:
- Competition closed / deadline passed: `bg-raw-steel/20 text-raw-steel` (matches existing Draft badge pattern)
- Competition full: same `bg-raw-steel/20 text-raw-steel`

```
min-h-screen bg-background px-8 py-12
  └── mx-auto max-w-[640px]            ← same max-width as competition creation form
        ├── Competition header          ← name (text-2xl font-semibold text-parchment) + date (text-sm text-raw-steel)
        ├── rounded-xl bg-card p-8 ring-1 ring-foreground/10
        │     └── form.space-y-6
        │           ├── Last Name field
        │           ├── First Name field  ← helper text below
        │           ├── Gender field      ← RadioGroup (Male / Female / Other)
        │           ├── Body Weight field ← number input, "kg" suffix in placeholder
        │           ├── Country field     ← Combobox (Popover + Command)
        │           ├── Events section    ← Checkbox per discipline; inline expand on check
        │           │     └── [if checked] Bell Weight select + Duration selector/text (inset pl-6 pt-4)
        │           ├── Club field        ← optional, helper text
        │           ├── Coach field       ← optional, helper text
        │           └── Submit button     ← full-width, patina-bronze
```

**Success page** (`/registration/[compId]/success?registrantId=xxx`):

```

  └── mx-auto max-w-[640px]
        ├── h1: "Registration Confirmed"   ← text-2xl font-semibold text-parchment
```

## Interaction Contracts

### Country Combobox (D-01, D-02, D-03)
- Trigger: native-styled button showing flag emoji + country name or "Select country" placeholder in `text-raw-steel/50`
- Popover opens below trigger, `w-[--radix-popover-trigger-width]` to match trigger width
- Command input: `placeholder="Search countries..."` — filters on name
- List items: flag emoji + country name, 16px text
- Selected state: checkmark icon (lucide `Check`) in `text-patina-bronze`
- Keyboard: Arrow Up/Down navigates, Enter selects, Escape closes

### Event Expand/Collapse (D-04)
- CSS transition: `max-height` from 0 to auto with `overflow-hidden transition-all duration-200 ease-in-out`
- Checked: expand area animates open (200ms ease-in-out)
- Unchecked: collapses immediately, clears bell weight + duration state
- The expand area has `pl-6` left inset to visually nest under checkbox

### Bell Weight Select (D-05)
- Native `<select>` element styled with `inputClass` (matching all other inputs)
- First `<option>` is disabled placeholder: `"Select bell weight"`
- Options populated from competition's `allowedBellWeights` filtered by event type (double for LC/Jerk, single for Snatch)

### Duration Selector (D-06)
- When `allowedDurations === 'both'`: `RadioGroup` with two items (10 min / 5 min), no default selected
- When `allowedDurations === '10'` or `'5'`: Static `<p>` text: `"Duration: 10 min"` or `"Duration: 5 min"` in `text-sm text-parchment`

### Form Validation
- Inline per-field validation (matching competition creation form pattern)
- Error text: `text-xs text-destructive mt-1` below the field (raw-steel color, existing pattern)
- Validation fires on submit attempt, not on blur
- Required fields: Last Name, First Name, Gender, Body Weight, Country, at least one Event
- Per-event validation: if event checked, bell weight must be selected; duration must be selected (when 'both')
- Submit button shows `disabled:opacity-50 disabled:cursor-not-allowed` while submitting
- Button label changes from "Register" to "Registering..." during submission

### Loading / Submitting State
- Submit button: `disabled` + text changes to "Registering..." (no spinner — matches competition creation form pattern)
- No page-level loading overlay

### Guard States
- `notFound()` (Next.js) for invalid compId — standard 404, no custom UI needed (D-10)
- Closed / deadline-passed / full: render guard card with message, no form rendered (D-09)
- Guard card is server-rendered — no client JS needed

## Accessibility Notes

- All form fields must have associated `<label>` or `aria-label`
- Combobox trigger must have `aria-haspopup="listbox"` and `aria-expanded` state
- Checkbox expand regions must use `aria-hidden="true"` when collapsed (or `display:none` equivalent)
- Color is never the only communicator of state — error messages always include text
- Country combobox: `role="combobox"` on trigger, `role="listbox"` on dropdown, `role="option"` on items
