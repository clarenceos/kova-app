Create `lib/actions/dashboard.ts` with `'use server'` directive. Three exported functions:

**1. `getCompetitionDashboard(compId: string)`**
Returns all data needed for the dashboard when a competition is selected.
- Return type: `Promise<{ competition: Competition; registrants: RegistrantWithEntries[]; totalCount: number } | { error: string }>`
- Where `RegistrantWithEntries` is: `Registrant & { entries: RegistrationEntry[] }`
- Export the `RegistrantWithEntries` type for use by client components
- Query: fetch competition by `eq(competitions.id, compId)`, return `{ error: 'Competition not found' }` if null
- Query: fetch all registrants where `eq(registrants.competitionId, compId)` ordered by `desc(registrants.createdAt)`
- Query: fetch all registration_entries where `eq(registrationEntries.competitionId, compId)`
- Group entries by registrantId in JS (not a join — Drizzle/libsql joins are verbose; 3 simple queries are cleaner for <200 registrants per D-09)
- Return `{ competition, registrants: registrantsWithEntries, totalCount: registrantsWithEntries.length }`

**2. `removeRegistrant(registrantId: string, competitionId: string)`**
Deletes a registrant and all their entries atomically.
- Return type: `Promise<{ success: true } | { error: string }>`
- Validate registrantId and competitionId are non-empty strings
- Use `db.batch([...])` with two delete statements:
  - `db.delete(registrationEntries).where(eq(registrationEntries.registrantId, registrantId))`
  - `db.delete(registrants).where(eq(registrants.id, registrantId))`
- Call `revalidatePath('/organizerdb')` after successful deletion
- Wrap in try/catch, return `{ error: 'Failed to remove registrant' }` on failure

**3. `bulkImportRegistrants(input: { competitionId: string; rows: CSVRow[] })`**
Bulk-creates registrants from validated CSV rows.
- Export `CSVRow` type: `{ lastName: string; firstName: string; gender: 'Male' | 'Female'; bodyWeightKg: number; country: string; events: Array<{ event: 'LC' | 'JERK' | 'SNATCH'; bellWeight: string; duration: number }>; club: string | null; coach: string | null }`
- Return type: `Promise<{ imported: number } | { error: string }>`
- Validate competitionId, fetch competition, check it exists
- For each row: generate a registrantId via `createId()`, generate serials via `generateCompetitionSerial(competitionId, comp.serialPrefix)` in a for-loop (one per event per row)
- Build a flat array of all insert statements: one `db.insert(registrants).values(...)` per row + one `db.insert(registrationEntries).values(...)` per event per row
- Execute all inserts via a single `db.batch([...allInserts])`
- Call `revalidatePath('/organizerdb')`
- Return `{ imported: rows.length }`

Also export a `getCompetitions()` helper function:
- Fetches all competitions ordered by `desc(competitions.createdAt)`
- Return type: `Promise<Competition[]>`
- Used by the server component to populate the selector

Import from drizzle-orm: `eq`, `desc`. Import from lib/schema: `competitions`, `registrants`, `registrationEntries` and their types. Import `createId` from `@paralleldrive/cuid2`. Import `generateCompetitionSerial` from `@/lib/queue/serial`. Import `revalidatePath` from `next/cache`.
