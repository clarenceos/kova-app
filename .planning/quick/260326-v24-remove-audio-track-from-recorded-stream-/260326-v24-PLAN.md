# Quick Task Plan: Remove Audio Track from Recorded Stream

## Context

Chrome's WebM encoder skips silent audio frames. With only a few beeps across a 60-second recording, the container has timestamps for those moments only, collapsing the video to ~2 seconds. Fix: record video-only, keep beeps as speaker-only output.

## Task 1: Remove audio mixing from recording pipeline

**Files:** `app/record/recording/page.tsx`
**Action:** 4 surgical edits:

1. Remove `audioDestRef` ref declaration (line 161)
2. Remove `audioDestRef.current` creation in `startSession` (lines 494-496)
3. Replace `combinedStream` with `canvasStream` in MediaRecorder (lines 519-523, 533)
4. Remove `audioDestRef.current` connections in `playBeep` (line 231) and `playStartTone` (line 259)

**Verify:** `grep -n audioDestRef app/record/recording/page.tsx` returns no matches
**Done:** MediaRecorder records canvas video only; beeps play through speakers only
