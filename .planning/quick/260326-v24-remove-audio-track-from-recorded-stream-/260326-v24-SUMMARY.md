# Quick Task 260326-v24: Remove Audio Track from Recorded Stream

## Problem
Chrome's WebM encoder skips silent audio frames from `MediaStreamAudioDestinationNode`. With only a few beeps across a 60-second recording, the container had timestamps for those moments only, collapsing the video to ~2 seconds.

## Changes

**File:** `app/record/recording/page.tsx`

1. Removed `audioDestRef` ref declaration
2. Removed `audioDestRef.current = audioCtxRef.current!.createMediaStreamDestination()` from `startSession`
3. Removed `combinedStream` — `MediaRecorder` now records `canvasStream` directly (video only)
4. Removed `gainNode.connect(audioDestRef.current)` from both `playBeep` and `playStartTone` — beeps play through speakers only
5. Updated MIME type detection to use video-only codecs (`vp9`/`vp8` without `opus`)

## Verification
- `grep audioDestRef` returns no matches
- TypeScript compiles clean
- Beeps still play through speakers via `ctx.destination`
- Video recording is video-only — no silent audio frames to cause timestamp skipping
