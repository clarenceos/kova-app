## Decisions Made
- IndexedDB save happens BEFORE React state update to ensure durable backup exists first
- 24-hour TTL on stored recordings to prevent stale data accumulation
- 1-second delayed redirect after IndexedDB miss to handle slow context propagation edge case
- Empty recording validation at 10KB threshold catches failed captures early with user-facing alert
- Stream cleanup moved to finally block so it always executes even on error
