## What Changed

### Bug 1 — 308 Resume (components/record/YouTubeUploader.tsx)
Restructured XHR upload into a resumable loop. On HTTP 308 (Resume Incomplete), reads the `Range` header (`bytes=0-N`), sets `startByte = N + 1`, slices the blob, adds `Content-Range` header, and retries. Progress tracking accounts for previously uploaded bytes. Only 4xx/5xx are treated as errors.

### Bug 2 — Save on failure (app/record/playback/page.tsx)
"Save video file" button now appears when upload fails (not just on success). Removed the `mimeType.includes('webm')` gate — all formats can be saved.

### Bug 3 — Upload Manually stays on page (app/record/playback/page.tsx)
Removed `router.push('/record/instructions')` navigation. Replaced with "Copy description" button that copies title + description to clipboard without leaving the page.

### Feature — Manual URL submission (app/record/playback/page.tsx)
When upload fails, shows:
1. "Save video file" button
2. "Copy description" button (with "Copied!" feedback)
3. Text input for YouTube URL + "Submit link" button

On submit: extracts video ID from URL, calls `createEntry` server action (same DB write path as auto-upload), clears IndexedDB. "Done — Go to Profile" button hidden until either auto-upload succeeds or manual URL submitted. Added TODO comment for Option B (`/profile/submit-link` page).
